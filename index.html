I built an app with you (or another vibe coding tool) and I want to rebuild it
in Zoho Creator — a low-code platform with proper roles, permissions, audit
trails, workflows, and native integrations. You already have the context of
this app from our previous conversation. Translate it into a Zoho Creator
spec I can use as a blueprint when I open Creator's app builder.

Output the spec in this exact structure, and nothing else:

## App overview
One-paragraph summary of what the app does and who uses it.

## Forms
For each form (each "table" or main entity in my current app), list:
- Form name
- Purpose (one line)
- Fields — for each field give: label, Zoho Creator field type
  (Single Line, Multi Line, Number, Decimal, Currency, Date, Date-Time,
  Email, Phone, Dropdown, Radio, Checkbox, Multi Select, Lookup, File Upload,
  Image, URL, Rich Text, Formula, Auto Number, Users, Subform), required (yes/no),
  and any default value or validation rule.

## Reports / Views
For each list/table view in my current app, list:
- Report name
- Source form
- Type (List View, Spreadsheet View, Calendar, Kanban, Map, Timeline,
  Pivot Table, Chart, Summary)
- Filters and sort order
- Which roles can see it

## Workflows
For each automation (email, status change, calculation, integration call), list:
- Workflow name
- Trigger (On Add, On Edit, On Delete, On Approval, Scheduled, Button click)
- Source form
- Action in plain English
- Deluge pseudocode for the action (real Deluge syntax where you can —
  use sendmail, info, openUrl, invokeurl, zoho.crm.create, etc.)

## Roles & permissions
- List the user roles in my app (Admin, Manager, Member, Customer, etc.)
- For each role: which forms/reports they can see, and which fields they can
  edit vs. view-only.

## Integrations
- List any external services the current app talks to (Stripe, Slack, Google
  Sheets, a CRM, etc.) and the Zoho Creator equivalent or the connector name.

## Build order
Numbered list of the order I should create things in Creator (forms first,
then lookups, then reports, then workflows, then roles), with any dependencies
called out.

Be specific. Use the actual data and logic from my app, not generic placeholders.
If anything is unclear, ask me one focused question at the end.
